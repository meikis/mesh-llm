Run large models across every machine you own

Mesh pools machines into a distributed inference network, so you can serve large models across multiple devices through one OpenAI-compatible endpoint.

Install Mesh Open Public Mesh Browse Catalog
quickstart 
# Install Mesh
curl -fsSL https://mesh-llm.cloud/install.sh | sh

# Join the public mesh and serve if this machine can
mesh-llm serve --auto

# Or join as an API-only client
mesh-llm client --auto

# Or serve a local model
mesh-llm serve --model Qwen3-8B

# Point agents at Mesh
export OPENAI_BASE_URL=http://localhost:3131/v1

One endpoint, many machines

Mesh handles discovery, placement, routing, and model serving so tools can talk to a single local API while the work happens across available machines.

Join or create a mesh

Start with one machine, join the public mesh, or create a private mesh for your team, lab, or home network.

Serve models locally

Use catalog names, Hugging Face GGUF URLs, or local model files. Mesh exposes an OpenAI-compatible endpoint.

Run beyond one device

Large models can use layer packages so multiple machines participate in one served model.
